## 1. Purpose & Thesis

Upgrade the MindForge meta-framework (npm `mindforge-cc`) from **v11.0.0 "Sovereign Stability"** to a "beast mode" next version. A 17-agent codebase recon (health score **82**, 41/43 tests passing) plus a 6-topic 2026 SOTA benchmark — cross-checked by an adversarial critic and verified directly against the repo — established that MindForge is **~70–75% real** (not the "60/40 stub" an initial pass implied). It is genuinely more substantial at the execution layer than most marketed agent frameworks.

The defining finding: **MindForge makes integrity and capability claims its code does not yet honor**, and **~40% of its surface reinvents native Claude Code primitives — some of it silently broken.**

**Thesis for this upgrade — "Beast Mode = make the claims true, then delegate the substrate."**
1. Make every *sovereign / tamper-evident / Merkle-linked / self-improving* claim literally true.
2. Stop reinventing the orchestration substrate; **align with native Claude Code / MCP / OTel** (user decision: *align-first*) and concentrate the ~20K-LOC budget on the differentiated layer native does **not** provide.
3. Close the sharp correctness gaps (dependency-DAG ordering, cost math) with discipline that never silently breaks existing users.

**Version strategy (user-approved, Approach A — "stabilize then build"):**
- **v11.0.1** — fast hygiene patch (pure fixes, no new capability).
- **v11.1.0** — beast-mode minor (4 pillars).
- **v12.0.0 (future)** — reserved for the "oceans": real post-quantum crypto/WebAuthn, full swarm-controller runtime, dense embeddings, default DAG ordering.

## 5. Architecture & Component Boundaries

Each unit has one purpose, a defined interface, and is independently testable.

- **UC-09 — write layer.** Single-writer append queue module (or WAL adapter) wrapping `knowledge-store`/`graph-edges`/`AUDIT` writes. Interface: `append(record) → Promise<void>` with ordered, fsync'd, rotation-aware semantics. Consumers unchanged at call sites.
- **UC-04 — audit signer/verifier.** `audit-writer` calls `signData(entry)`; `verify-audit` walks `previous_hash` + signature chain, exits non-zero on break. Interface boundary: signing is a pure function of (entry, DID key); verification is read-only.
- **UC-03 — wave planner.** `planWaves(tasks, {useDag})` returns dependency-respecting layers when flagged, falls back to `.wave`/single-wave otherwise. Cycle detection is a pre-flight gate that throws. File-conflict detection groups same-surface writers serially.
- **UC-10 — council runtime.** `runCouncil(question, {voices, threshold}) → {verdict, consensus, dissent[]}`. Reuses Semaphore + model router. Writes decision record; no other side effects.
- **UC-05 — PricingRegistry.** `price(modelId, {input, output, cacheRead, cacheCreate}) → usd`. Single load from config; all providers + cost-tracker depend on it; nothing hardcodes rates.
- **UC-06 — difficulty scorer.** `score(task) → 1..10` (pure heuristic); router maps score+tier→model with a Tier-3 floor. Shadow-mode = log intended vs actual, no behavior change.
- **UC-19 — native alignment.** `.claude/settings.json` hook registration; `.claude/agents/*.md` generated from personas; plugin manifest. Boundary: native runtime owns dispatch/isolation; MindForge owns the differentiated overlay (memory/cost/audit) exposed via MCP.
- **UC-18 — OTLP exporter.** `export(span) → OTLP` translation; flagged, additive, no change to local AUDIT sink.
- **UC-20 — retrieval fusion.** `fuse(rankedLists[]) → RRF order`, then `rerank(topN)` via model router. Replaces 3 incomparable linear blends behind one `query()` facade.

## 6. Error Handling & Safety Principles

- **Fail closed on integrity:** version mismatch (UC-01), broken audit chain (UC-04), unknown Tier-3 path (UC-22) → halt/escalate, never silently proceed.
- **Fail loud on correctness:** dependency cycles (UC-03) HALT at pre-flight with the cycle path, never warn-and-continue.
- **Shadow before gating:** UC-06 routing and any UC-20/UC-06 default flip log intended behavior for N runs before they take effect; UC-25 eval must show no regression first.
- **Opt-in for behavior changes:** UC-03 DAG ordering is flagged; explicit `.wave` always wins so existing PLANs don't silently reorder.
- **No theater:** UC-24 keeps the honest "simulated" label and removes simulated crypto from the live trust path rather than faking real crypto.

## 7. Testing Strategy

- **TDD-first per UC** (Red→Green→Refactor), native `assert` harness, mirror existing `tests/` style.
- **UC-00 gate (blocking, runs first in 11.1.0):** test-integrity audit — confirm the suite exercises real `bin/` runtime paths, not test-local reimplementations (critical because DAG/council logic currently lives only in test files). Output: a short report flagging any green-washed tests; fix before building on them.
- **Regression tests for the "lies":** UC-07a asserts a real child process ran and produced output; UC-01 runs the migration and asserts `config.json` reaches target version.
- **Coverage ratchet:** raise the CI gate stepwise (30→40→50→toward 60 lines / 40 branches) across the release — never one jump (avoids blocking feature PRs and coverage theater).
- **Eval harness (UC-25):** versioned golden set — query→expected entry IDs (recall@k/nDCG/MRR) for retrieval; golden prompts + rubric (LLM-as-judge) for routing — wired into the unified runner so cost-vs-quality is measurable before defaults flip.
- **Adversarial review per pillar:** a skeptic agent stress-tests each pillar's diff for the failure modes named in §6 before merge.

## 10. Key Risks & Mitigations

| Risk | Mitigation |
|---|---|
| Signing a lossy/lock-free log → broken chains | Hard ordering UC-09 → UC-04 |
| DAG silently reorders existing PLANs | Opt-in flag; `.wave` override; cycle HALT loud |
| Difficulty router degrades quality to save pennies | Shadow-mode + Tier-3 floor + eval gate (UC-25) |
| Coverage jump blocks feature PRs / breeds hollow tests | Ratchet stepwise |
| Tests assert fiction (green-washing) | UC-00 blocking audit first |
| Native alignment breaks a working (if dead) path | Phase UC-19 a→b→c; keep differentiated overlay intact; MCP seam, not rewrite |
| Scope creep into oceans | UC-13/16/17 + default DAG explicitly parked for v12.0.0 |
