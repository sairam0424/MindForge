The scripts themselves (`mindforge-context-monitor.js`, `mindforge-prompt-guard.js`) are well-written and already speak the native stdin JSON contract. Fix: create `.claude/settings.json` with the correct event names.

Verify by reading the Claude Code hook documentation or the existing hook format in any reference.

### UC-20 — RRF fusion over 3 retrieval paths + LLM-as-reranker

The recon found 3 retrieval paths that never fuse and use incomparable scoring: BM25 `textScore*0.7+confidence*0.3` (knowledge-indexer), TF-IDF+graph `embeddingScore*0.6+graphScore*0.4` (knowledge-graph), and SQLite FTS4 (vector-hub). Replace with scale-free Reciprocal Rank Fusion.

**Task 4:**
- [ ] Create `bin/memory/retrieval-fusion.js` — `fuseResults(rankedLists[]) → RRF-ordered results`. RRF formula: `score(item) = Σ 1/(k+rank)` where k~60 (standard). Each ranked list is `[{id, score}]` ordered by descending score; the function converts to rank-based RRF scores and merges/dedupes by id.
- [ ] Optionally: after RRF, the top-N results can be re-ranked via an LLM call (reuse the model router + anthropic provider — query the model "which of these is most relevant to <query>?"). This is expensive; gate behind a config flag `memory.rerank_enabled` (default false for now). Without re-ranking, pure RRF is still a major improvement over the incomparable linear blends.
- [ ] Wire into `bin/memory/auto-shadow.js` `getShadowContext()` — instead of its current single-path scoring, call all 3 retrievers, then fuse. Keep the existing retriever calls unchanged; just merge their outputs via RRF before selecting the top items for injection.
- [ ] Add tests: (a) RRF fusion of 3 lists produces expected merged order; (b) an item appearing in multiple lists ranks higher than one in only one list.
- [ ] Commit: `git commit -m "feat(memory): RRF fusion over 3 retrieval paths replacing incomparable linear blends (UC-20)"`
