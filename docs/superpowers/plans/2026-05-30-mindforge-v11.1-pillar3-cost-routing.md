## Unit clarification (important for UC-05)
- Provider hardcodes: `0.000003` = per-TOKEN = $3/1M tokens.
- Config `market_registry`: `0.003` = per-1K-tokens = $3/1M tokens (same dollars, different unit).
- Both resolve to the same $/1M — the "scale bug" was a unit-label mismatch, not a 1000x error in actual pricing. UC-05 must pick ONE unit (recommend **per-1M-tokens** as the canonical stored unit — it's human-readable: "$3/1M" is clearer than "$0.000003/token") and provide a `pricePerCall(model, {input,output,cacheRead,cacheCreate})` that handles conversion.

- [ ] **Step 2 (GREEN): Implement `bin/models/pricing-registry.js`**

```javascript
'use strict';
/**
 * MindForge — Pricing Registry (UC-05).
 * Single source of truth for model pricing. Loads from config.json market_registry.
 * All prices stored as USD per 1M tokens (human-readable).
 * Conversion: per-call cost = (tokens / 1_000_000) * price_per_1M.
 */
const fs = require('fs');
const path = require('path');

const FALLBACK_PRICE_PER_1M = { input: 3.0, output: 15.0, cache_read: 0.3, cache_creation: 3.75 };
let _registry = null;

function loadRegistry() {
  if (_registry) return _registry;
  try {
    const configPath = path.join(process.cwd(), '.mindforge', 'config.json');
    const config = JSON.parse(fs.readFileSync(configPath, 'utf8'));
    const raw = config.revops?.market_registry || {};
    _registry = {};
    for (const [modelId, entry] of Object.entries(raw)) {
      _registry[modelId] = {
        input: (entry.cost_input || 0) * 1000,
        output: (entry.cost_output || 0) * 1000,
        cache_read: (entry.cost_cache_read || entry.cost_input * 0.1) * 1000,
        cache_creation: (entry.cost_cache_creation || entry.cost_input * 1.25) * 1000,
      };
    }
  } catch {
    _registry = {};
  }
  return _registry;
}

function getPrice(modelId, bucket = 'input') {
  const reg = loadRegistry();
  const entry = reg[modelId];
  if (!entry) {
    process.stderr.write(`[pricing-registry] unknown model "${modelId}" — using fallback\n`);
    return FALLBACK_PRICE_PER_1M[bucket] || FALLBACK_PRICE_PER_1M.input;
  }
  return entry[bucket] || FALLBACK_PRICE_PER_1M[bucket];
}

function priceCall(modelId, usage = {}) {
  const input = (usage.input_tokens || 0) / 1_000_000 * getPrice(modelId, 'input');
  const output = (usage.output_tokens || 0) / 1_000_000 * getPrice(modelId, 'output');
  const cacheRead = (usage.cache_read_input_tokens || 0) / 1_000_000 * getPrice(modelId, 'cache_read');
  const cacheCreate = (usage.cache_creation_input_tokens || 0) / 1_000_000 * getPrice(modelId, 'cache_creation');
  return input + output + cacheRead + cacheCreate;
}

function clearCache() { _registry = null; }

module.exports = { getPrice, priceCall, clearCache };
```

## UC-21 — Prompt-cache accounting (~90% savings)

### Task 2: Anthropic provider sends cache_control + parses cache usage

- [ ] **Step 1: Modify `bin/models/anthropic-provider.js` `complete()`**

In the request body, add `cache_control: { type: 'ephemeral' }` to the system message content block (the `system` field). This tells the API to cache the system prompt across calls. The API response's `usage` object will then include `cache_read_input_tokens` and `cache_creation_input_tokens` alongside `input_tokens`/`output_tokens`.

Parse those fields from the response:
```javascript
const cacheRead = json.usage.cache_read_input_tokens || 0;
const cacheCreate = json.usage.cache_creation_input_tokens || 0;
```

Pass them to `priceCall`:
```javascript
const cost = priceCall(json.model, {
  input_tokens: inputTokens,
  output_tokens: outputTokens,
  cache_read_input_tokens: cacheRead,
  cache_creation_input_tokens: cacheCreate,
});
```

Include them in the response object:
```javascript
resolve({
  model: json.model,
  content: json.content[0].text,
  input_tokens: inputTokens,
  output_tokens: outputTokens,
  cache_read_input_tokens: cacheRead,
  cache_creation_input_tokens: cacheCreate,
  cost_usd: cost,
  provider: 'anthropic'
});
```

- [ ] **Step 2 (GREEN): Implement `bin/models/difficulty-scorer.js`**

A pure heuristic scorer. Factors: description length/complexity keywords, file count, Tier-3 floor.

```javascript
'use strict';
/**
 * MindForge — Difficulty Scorer (UC-06). Heuristic 1-10.
 * Used by model-router in SHADOW MODE to log what it WOULD route.
 */

const HIGH_KEYWORDS = /auth|jwt|oauth|crypto|security|payment|pii|gdpr|hipaa|encrypt|secret|credential/i;
const MEDIUM_KEYWORDS = /refactor|migrate|architect|design|performance|concurrency|async/i;

function score(task = {}) {
  const desc = task.description || '';
  const files = task.files || [];
  const tier = task.tier || 0;
  let s = 3; // baseline

  if (HIGH_KEYWORDS.test(desc) || files.some(f => HIGH_KEYWORDS.test(f))) s += 4;
  else if (MEDIUM_KEYWORDS.test(desc)) s += 2;

  if (files.length > 10) s += 2;
  else if (files.length > 5) s += 1;

  if (desc.length > 500) s += 1;

  // Tier-3 floor: auth/payment/PII tasks never route to a cheap model
  if (tier >= 3) s = Math.max(s, 7);

  return Math.min(Math.max(s, 1), 10);
}

module.exports = { score };
```
